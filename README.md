# PiP カンペツール

GitHub Pagesで公開する、画像カンペ用の静的Webアプリです。

## 公開URL

- アプリ: https://linn-0412.github.io/pip-kanpe-tool/
- リポジトリ: https://github.com/Linn-0412/pip-kanpe-tool

## 機能

- 画像を最大80枚まで登録
- グループ名を付けて画像を分類し、表示/PiP対象を絞り込み
- 登録後の画像名変更
- Document Picture-in-Pictureで常時前面の小窓表示
- PiP内の左右ボタンで前後切り替え
- PiP内のコントロール表示位置、サイズ、背景の濃さ、自動非表示を調整できます
- サムネイル画像は遅延読み込みと非同期デコードで、一覧表示の負荷を抑えます
- 画像は各ユーザーのブラウザ内IndexedDBに保存
- サーバー側には画像を保存しません
- 初回アクセス時に使い方ガイドを表示し、次回以降の非表示設定を保存できます
- Chrome/Edge拡張機能を併用すると、グローバルショートカットで前後の画像に切り替えできます

## 使い方

1. ChromeまたはEdgeでページを開く
2. カンペ画像を登録する。複数ファイルの同時選択に対応しており、ファイル名順で追加されます。
3. `PiPで表示` を押す
4. PiP小窓の `←` / `→` で切り替える

大量に登録する場合は、登録前に `保存時に長辺2048pxへ軽量化` をオンにしておくと、保存容量とサムネイル一覧の負荷を抑えやすくなります。

使い方ガイドはヘッダーの `使い方` ボタンから再表示できます。URL末尾に `?guide=1` を付けても強制表示できます。

FF14でゲーム上に重ねる場合の表示可否は、ゲーム内表記に合わせると次の通りです。

- フルスクリーン: 表示不可
- 仮想フルスクリーン: 表示可能
- ウィンドウモード: 表示可能

## Chrome/Edge拡張機能

Releaseに添付している拡張機能ZIPを使うと、ゲームをアクティブにしたままショートカットでカンペを切り替えられます。

1. [こちらから拡張機能ZIPをダウンロードする](https://github.com/Linn-0412/pip-kanpe-tool/releases/latest/download/pip-kanpe-tool-hotkeys-extension.zip)
2. ZIPを任意の場所に解凍する
3. 使っているブラウザに合わせて、次の手順で拡張機能を読み込む

### Chromeで使う

1. Chromeの拡張機能画面（`chrome://extensions/`）を開く
2. 右上の `デベロッパーモード` をオンにする
3. `パッケージ化されていない拡張機能を読み込む` から、解凍したフォルダを選ぶ
4. ショートカット設定画面（`chrome://extensions/shortcuts`）を開く
5. `PiP カンペツール Hotkeys` のショートカットを確認する

### Edgeで使う

1. Edgeの拡張機能画面（`edge://extensions/`）を開く
2. 左側または右側に表示される `開発者モード` をオンにする
3. `展開して読み込み` または `パッケージ化されていない拡張機能を読み込む` から、解凍したフォルダを選ぶ
4. ショートカット設定画面（`edge://extensions/shortcuts`）を開く
5. `PiP カンペツール Hotkeys` のショートカットを確認する

初期設定では `Ctrl+Shift+9` が次、`Ctrl+Shift+8` が前です。FF14のキーバインドと衝突する場合は、各ブラウザのショートカット設定画面で別のキーに変更してください。ブラウザの仕様上、環境によってはショートカット画面で手動設定が必要です。

拡張機能を更新したあとは、Chromeは `chrome://extensions/`、Edgeは `edge://extensions/` の更新ボタンを押すか、拡張機能を読み込み直してください。PiPカンペツールのページも再読み込みすると確実です。

開発者向けには、このリポジトリ内の `extension/` フォルダを直接読み込むこともできます。

※ ローカル環境で動作確認する場合は、`index.html` を直接ブラウザで開くのではなく、`python -m http.server 8000` などで `http://localhost:8000/index.html` として開いてください。`file://` 直開きだとPiP用のCSSがうまく読み込めず、表示が崩れることがあります。

## 注意

Document Picture-in-Picture APIは対応ブラウザが限られます。非対応ブラウザでは通常プレビューのみ利用できます。
